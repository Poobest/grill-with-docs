# Phase 0 — Tenant App UI Design Contract

> Visual and interaction contract for the multi-tenant Installment-financing SaaS **Tenant web app** (`apps/tenant`). Generated by gsd-ui-researcher, verified by gsd-ui-checker. Platform admin app and LINE LIFF customer channel are out of scope here, but all tokens are chosen so they can be reused by them later.

## Spacing Scale

Declared values (must be multiples of 4). This is a **compact, data-dense back-office tool** — defaults lean to the smaller end of the scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner padding, inline label-to-input gap |
| sm | 8px | Default compact element spacing, table cell vertical padding, form field row gap |
| md | 16px | Card padding, default gap between form groups, table cell horizontal padding |
| lg | 24px | Section padding, gap between KPI stat cards |
| xl | 32px | Layout gaps, page content top padding |
| 2xl | 48px | Major section breaks on long detail pages |
| 3xl | 64px | Page-level outer spacing (rare; sidebar-to-content max gutter) |

Default element spacing lives at **sm (8px) / md (16px)** for this app — `lg`+ is reserved for layout-level separation only.

Exceptions:
- **Touch targets:** Icon-only buttons (row actions in tables, approve/reject slip controls) have a minimum hit area of **40px × 40px** even when the visible glyph is smaller, to stay tappable on tablets used at the counter.
- **Data table row height:** **40px** compact rows (not on the 4-grid for the glyph, but the 40px box is a multiple of 4). This is the intentional density choice.

## Typography

Compact scale tuned for dense tables/forms. Body is ~14px.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 (21px) |
| Label | 13px | 600 (semibold) | 1.4 (18px) |
| Heading | 20px | 600 (semibold) | 1.3 (26px) |
| Display | 28px | 600 (semibold) | 1.2 (34px) |

Only **two weights** are used across the app: **400 (regular)** for body/values and **600 (semibold)** for labels, headings, table column headers, KPI numbers, and emphasis. No light, medium, or bold-900 weights.

Usage map:
- **Display (28/600):** KPI card primary numbers (ยอดขายรวม, commission รวม), dashboard page title.
- **Heading (20/600):** Section headings, dialog titles, detail-page titles (สัญญา #, ลูกค้า).
- **Label (13/600):** Form field labels, table column headers, badge text, KPI card captions.
- **Body (14/400):** Table cell values, form input text, paragraph copy, money amounts (tabular-nums), installment schedule rows.

## Color

Light mode. 60/30/10 split. Accent is **Emerald/Teal green** reserved strictly for the 10% role.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (white) / page canvas `#F8FAFC` (slate-50) | App background, content canvas, table body, input fields |
| Secondary (30%) | `#F1F5F9` (slate-100) surfaces, `#E2E8F0` (slate-200) borders, `#0F172A` (slate-900) primary text, `#64748B` (slate-500) muted text | Sidebar, nav, cards, table header row, dividers, secondary text |
| Accent (10%) | `#059669` (emerald-600), hover `#047857` (emerald-700) | See "Accent reserved for" — interactive emphasis only |
| Destructive | `#DC2626` (red-600), hover `#B91C1C` (red-700) | Destructive/financial-risk actions only |

Accent reserved for:
- Primary buttons (สร้างสัญญา, บันทึกการรับเงิน, อนุมัติ, บันทึก)
- Text links and clickable IDs (เลขที่สัญญา, เลขที่ใบเสร็จ)
- Active/selected sidebar nav item (background tint `#ECFDF5` emerald-50 + emerald-600 text/indicator)
- Focused & selected form-control state (input focus ring, selected table row, active tab underline, checked checkbox/radio)

Accent is **NOT** used for: all body text, all borders by default, secondary/ghost buttons, table chrome, or "every interactive element." Secondary buttons use slate-200 surface + slate-900 text.

**Status badge palette** (semantic, used in tables — these are status tokens, not part of the 10% accent budget):

| Domain status | Color | Hex |
|---------------|-------|-----|
| Contract `ACTIVE`, Payment `APPROVED`, Contract `COMPLETED` | emerald | `#059669` on `#ECFDF5` |
| Contract `PENDING_DOWN_PAYMENT`, Payment `PENDING` | amber | `#D97706` on `#FFFBEB` |
| Contract `DEFAULTED`, Payment `REJECTED` | red | `#DC2626` on `#FEF2F2` |
| Contract `CANCELLED` | slate (neutral) | `#475569` on `#F1F5F9` |

## Visual Hierarchy

What draws the eye first on each primary screen — removes executor guesswork on focal points.

| Screen | Focal point (1st) | 2nd | 3rd |
|--------|-------------------|-----|-----|
| Dashboard | KPI **Display numbers** (28/600) in stat cards | KPI captions (Label 13/600) | Arrears / collect-today table below |
| List pages (สัญญา / ลูกค้า / payments) | Primary CTA top-right (สร้างสัญญา) in accent emerald | First data column + status **badge** color | Row actions (right-aligned, muted until hover) |
| Detail page (สัญญา #) | Page title + status badge | Money summary (ยอดรวม / คงเหลือ, tabular-nums) | Installment schedule table |
| Record-payment / approve-slip dialog | Amount + งวดที่ {n}/{total} | Primary accent confirm button | Slip image / context |

Rule: exactly **one** accent-colored primary action per screen view; everything else competes through type weight and badge color, not additional accent fills.

**Accessibility for icon-only controls:** Every icon-only control (table row actions, approve/reject slip buttons, row overflow menu) MUST carry both a `tooltip` (shadcn-vue `tooltip` block) and an `aria-label` in Thai (e.g. `aria-label="อนุมัติสลิป"`, `"ปฏิเสธสลิป"`, `"แก้ไข"`, `"ยกเลิกสัญญา"`). Never ship an icon button with no accessible name.

## Copywriting Contract

All UI copy is **Thai**. Concise English appears in parentheses only where it genuinely aids recognition.

| Element | Copy |
|---------|------|
| Primary CTA — create contract | สร้างสัญญา |
| Primary CTA — create customer | เพิ่มลูกค้า |
| Primary CTA — record cash payment | บันทึกการรับเงิน |
| Primary CTA — approve slip | อนุมัติสลิป |
| Primary CTA — generic save | บันทึก |
| Empty state heading | ยังไม่มีสัญญา |
| Empty state body | เริ่มต้นด้วยการสร้างสัญญาผ่อนชำระให้ลูกค้า แล้วระบบจะคำนวณตารางงวดให้อัตโนมัติ — กดปุ่ม "สร้างสัญญา" เพื่อเริ่ม |
| Error state | ไม่สามารถโหลดข้อมูลได้ กรุณาตรวจสอบการเชื่อมต่อแล้วลองใหม่อีกครั้ง (ลองใหม่) |
| Destructive confirmation | ยกเลิกสัญญา: ยืนยันการยกเลิกสัญญานี้หรือไม่? ระบบจะคืน stock สินค้าเข้าสาขาโดยอัตโนมัติ และไม่สามารถย้อนกลับได้ — ปุ่มยืนยัน "ยกเลิกสัญญา" (สีแดง), ปุ่มยกเลิก "ไม่ใช่ตอนนี้" |

Additional locked copy patterns (shared across screens):
- **Other empty states:** ยังไม่มีลูกค้า / ยังไม่มีสินค้า / ไม่มีรายการเก็บเงินวันนี้ / ยังไม่มีรายการค้างชำระ — same heading+body shape (สั้น + บอก next step).
- **Other destructive/financial confirmations** (all use red confirm button + neutral cancel "ยกเลิก"):
  - ระงับลูกค้า: ยืนยันการระงับลูกค้ารายนี้? ลูกค้าจะไม่สามารถสร้างสัญญาใหม่ได้จนกว่าจะปลดระงับ
  - ปฏิเสธสลิป: ยืนยันการปฏิเสธสลิปการโอนเงินนี้? งวดนี้จะยังคงค้างชำระ
- **Success toasts (sonner):** บันทึกการรับเงินเรียบร้อย ออกใบเสร็จเลขที่ {receiptNumber} / สร้างสัญญาเรียบร้อย / อนุมัติการชำระเงินแล้ว.
- **Field/format conventions:** money formatted as `฿1,234` (THB, thousands separator, no decimals for whole baht); installment labels as `งวดที่ {n}/{total}`; due date label `วันครบกำหนด`.

## Registry Safety

shadcn-vue is not yet initialized; these are the **official shadcn-vue blocks** expected to be added (via `npx shadcn-vue add ...`) when the design system is initialized in a later task. No third-party registries are declared.

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn-vue official | button, input, label, textarea, select, checkbox, radio-group, form, table, card, badge, dialog, alert-dialog, dropdown-menu, tabs, sonner (toast), skeleton, separator, sheet, tooltip, pagination, popover, calendar / date-picker | not required |
| (third-party) | none | not applicable |

Notes:
- **alert-dialog** is the required pattern for every destructive/financial confirmation listed in the Copywriting Contract (ยกเลิกสัญญา, ระงับลูกค้า, ปฏิเสธสลิป).
- **sonner** is the toast surface for receipt/commission/approval success messages.
- **table** + **pagination** + **badge** form the core data-dense list pattern (contracts, payments, customers, arrears, collect-today).
- **card** is the KPI stat-card surface on the dashboard.
- If a later task introduces ANY third-party registry, the registry vetting gate (`npx shadcn-vue view {block} --registry {url}` + flagged-pattern scan) MUST run before that block enters this contract.
