ระบบบริหารจัดการสินเชื่อผ่อนชำระ

Multi-tenant SaaS สำหรับร้านค้าที่ขายสินค้าผ่อนชำระรายวัน/สัปดาห์/เดือน

🎯 Problem Statement & Solution

ร้านค้าขายสินค้าทนทาน (เช่น อิเล็กทรอนิกส์) ที่ให้ลูกค้าผ่อนชำระรายวัน/รายสัปดาห์/รายเดือน ยังไม่มีระบบดิจิทัลที่รองรับกระบวนการทั้งหมด ตั้งแต่การสร้างสัญญาผ่อน การติดตามการชำระเงิน การคำนวณ commission ให้ Sale และการแจ้งเตือนลูกค้า ร้านส่วนใหญ่ยังใช้ Excel หรือกระดาษบันทึก ทำให้เกิดข้อผิดพลาด ข้อมูลสูญหาย และไม่สามารถขยายสาขาได้อย่างมีประสิทธิภาพ
Platform Admin

จัดการ Tenant และ Subscription plan ทั้งหมด

Tenant Admin

จัดการร้านของตัวเอง ทั้ง Branch, User, Product, Stock

Sale Lead / Sale

สร้างสัญญา เก็บเงิน ออกใบเสร็จ รับ commission

Customer

ดูสัญญาผ่อนและชำระผ่าน LINE LIFF โดยไม่ต้องโหลด app

📋 User Stories

Platform Admin 6 stories
1
Platform Admin ต้องการสร้าง Subscription Plan ที่กำหนดจำนวนสาขาสูงสุดและจำนวน user สูงสุดได้ → ให้ Tenant เลือก plan ที่เหมาะสมกับขนาดธุรกิจ
2
Platform Admin ต้องการสร้าง Tenant ใหม่พร้อมกำหนด plan ได้ → onboard ร้านค้าใหม่เข้าสู่ระบบ
3
Platform Admin ต้องการดูรายการ Tenant ทั้งหมด พร้อมสถานะ subscription และวันหมดอายุ → ติดตาม account ที่ใกล้หมดอายุ
4
Platform Admin ต้องการ extend subscription ของ Tenant ด้วยตัวเองหลังยืนยันการโอนเงิน → เปิดการใช้งานต่อ
5
Platform Admin ต้องการ lock Tenant ที่ subscription หมดอายุโดยอัตโนมัติ → ไม่ให้ใช้งานโดยไม่ชำระค่าบริการ
6
Platform Admin ต้องการ login ผ่าน Platform app แยกต่างหากจาก Tenant app → แยก session และ permission อย่างชัดเจน
Tenant Admin 11 stories
7
Tenant Admin ต้องการสร้างสาขาใหม่ได้ภายในขีดจำกัดของ plan → ขยายธุรกิจ
8
Tenant Admin ต้องการสร้าง User ใหม่ในระบบ พร้อมกำหนด role (Sale Lead / Sale) และสาขาที่สังกัด → ให้พนักงานเข้าถึงระบบได้
9
Tenant Admin ต้องการเพิ่มสินค้าพร้อมกำหนดราคาแยกตามประเภท (เงินสด / รายวัน / รายสัปดาห์ / รายเดือน) และกำหนดเงินดาวน์ → ให้ Sale สร้างสัญญาได้ถูกต้อง
10
Tenant Admin ต้องการกำหนด commission rate ให้ Sale แต่ละคนแยกระหว่างยอดผ่อนและยอดเงินสด → ให้ Sale มี incentive ติดตามหนี้
11
Tenant Admin ต้องการกำหนด commission rate สำหรับ Sale Lead แบบ override commission → ให้ Sale Lead ดูแลทีมอย่างมีประสิทธิภาพ
12
Tenant Admin ต้องการตั้ง stock สินค้าแต่ละสาขาได้ → ควบคุมปริมาณสินค้าคงเหลือ
13
Tenant Admin ต้องการกำหนด contract limit ให้ลูกค้าแต่ละคน → ควบคุมความเสี่ยงในการให้สินเชื่อ
14
Tenant Admin ต้องการ suspend ลูกค้าที่ค้างชำระ → ป้องกันการสร้างสัญญาใหม่จนกว่าจะชำระหนี้ครบ
15
Tenant Admin ต้องการยกเลิกสัญญาผ่อนได้ พร้อมให้ระบบคืน stock อัตโนมัติ → จัดการกรณีพิเศษที่ Sale ทำไม่ได้
16
Tenant Admin ต้องการดู dashboard ยอดขายรวมทุกสาขา commission ทั้งหมด stock ทุกสาขา และรายการลูกค้าค้างชำระ → ตัดสินใจเชิงธุรกิจได้รวดเร็ว
17
Tenant Admin ต้องการดูรายงานแยกประเภทผ่อน daily/weekly/monthly → วิเคราะห์ portfolio สินเชื่อ

⚙️ Implementation Decisions

Backend Modules (NestJS)

Moduleสถานะหมายเหตุ
Auth (Tenant + Platform)สมบูรณ์JWT dual-secret
Platform Plansสมบูรณ์CRUD
Platform Tenantsสมบูรณ์CRUD + subscription extend
Branchesสมบูรณ์CRUD
Usersสมบูรณ์CRUD + role management
Productsสมบูรณ์5 price fields: cashPrice, downPayment, dailyPrice, weeklyPrice, monthlyPrice
Stockสมบูรณ์set / adjust / decrement-on-down-payment / increment-on-cancel
Customersสมบูรณ์suspend / unsuspend / set contract limit
Contractsสมบูรณ์create ด้วย transaction (contract + schedule), cancel, document endpoint
Paymentsสมบูรณ์cash record, slip submit, approve, reject, receipt endpoint
Commissionsสมบูรณ์คำนวณจากยอดเก็บจริง (ADR-0002)
Jobs (Late Fee + Notification)ร่างแล้วProcessor มีอยู่ แต่ logic ยังไม่สมบูรณ์

การคำนวณ Commission (ADR-0002)

Commission คำนวณเมื่อ Payment ถูก approve เท่านั้น (ไม่ใช่วันสร้างสัญญา)
Sale: payment.amount × sale.commissionRate
Sale Lead: payment.amount × saleLead.overrideCommissionRate (คำนวณจาก Sale ทุกคนในทีม)
Rate แยกระหว่าง installmentRate และ cashRate ตาม paymentType

Schedule ที่ระบบสร้างอัตโนมัติ

DAILY:   30 งวด × dailyPrice   + downPayment
WEEKLY:  12 งวด × weeklyPrice  + downPayment
MONTHLY: 12 งวด × monthlyPrice + downPayment
CASH:     1 งวด × cashPrice    (ไม่มี downPayment แยก)

📐 Architecture Decision Records

ADR-0001

Vue + NestJS + PostgreSQL

เลือกเพราะ team มีประสบการณ์ NestJS และ PostgreSQL รองรับ complex report query

ADR-0002

Commission คำนวณจากยอดเก็บจริง ไม่ใช่ยอดขาย

ทำให้ Sale มี incentive ติดตามหนี้ แทนที่จะได้ commission ทันทีที่สร้างสัญญา

ADR-0003

Row-level multi-tenancy ด้วย tenant_id

ง่ายกว่า schema-per-tenant และ Prisma support ดีกว่า NestJS inject tenantId อัตโนมัติทุก query

ADR-0004

Manual subscription payment

คุ้มกว่า payment gateway สำหรับจำนวน Tenant น้อย — Platform Admin extend หลัง Tenant โอนเงิน

Stock timing note: Stock ลดตอน approve เงินดาวน์ (ไม่ใช่ตอนสร้างสัญญา) เพราะสินค้าออกจากร้านเมื่อลูกค้าชำระเงินดาวน์แล้วเท่านั้น
Defaulted threshold: กำหนดใน tenant.defaultThreshold เป็นจำนวนวันค้างชำระ — Admin ปรับได้ต่อ Tenant
